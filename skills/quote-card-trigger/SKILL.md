# Quote Card Trigger

## 适用场景

- 输入完全由单个 Markdown 引用块组成，需要确认是否触发写卡。
- 只负责“触发判定 + 把原文交给 add-card”，不在本技能内做摘要或写入。

## 操作流程（仅检测）

1. **复制引用原文**：保留每行开头的 `>` 方便校验。  
2. **验证结构**：所有非空行都必须以 `>` 开头；若混入正文或出现多个引用块，提示用户拆分后再触发。  
3. **确认意图**：确保用户希望将该引用内容记录到闪念卡。  
4. **交给 add-card**：将整段原始引用文本（可附带用户给出的标签/额外提示）传递给 `add-card` 技能，由它负责抽取标题、摘要并调用脚本写卡。  
5. **反馈结果**：把 add-card 返回的路径与摘要转述给用户。

## 注意事项

- 如果引用块内含多段内容，不触发。
- 若引用前后夹杂未加 `>` 的普通文本（如补充说明、上下文等），视为结构不纯，不触发；先请用户只提供引用内容。  
- `quote-card-trigger` 不做摘要和标题；这些步骤由 `add-card` 负责，确保两技能职责清晰。  
- 保证 `add-card` 可用；如写卡失败，提示用户稍后再试或改用其他方式。
